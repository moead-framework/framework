moead_framework.algorithm.combinatorial.moead_sps_random.MoeadSPSRandom¶

class moead_framework.algorithm.combinatorial.moead_sps_random.MoeadSPSRandom(problem, max_evaluation, number_of_weight_neighborhood, number_of_subproblem_to_visit, aggregation_function, weight_file, number_of_objective=None, number_of_crossover_points=2, mating_pool_selector=None, genetic_operator=None, parent_selector=None)[source]¶

Bases: moead_framework.algorithm.combinatorial.moead.Moead

Implementation of MOEA/D with the component SpsRandomAndBoundaries.

__init__(problem, max_evaluation, number_of_weight_neighborhood, number_of_subproblem_to_visit, aggregation_function, weight_file, number_of_objective=None, number_of_crossover_points=2, mating_pool_selector=None, genetic_operator=None, parent_selector=None)[source]¶

Constructor of the algorithm.

Parameters

  • problem – {Problem} problem to optimize

  • max_evaluation – {integer} maximum number of evaluation

  • number_of_objective – {integer} number of objective in the problem

  • number_of_weight_neighborhood – {integer} size of the neighborhood

  • number_of_subproblem_to_visit – {integer} number of sub-problems to visit randomly at each generation

  • weight_file – {string} path of the weight file. Each line represent a weight vector, each column represent a coordinate. An exemple is available here: https://github.com/moead-framework/data/blob/master/weights/SOBOL-2objs-10wei.ws

  • aggregation_function – {AggregationFunction}

  • genetic_operator – Optional – {GeneticOperator} The default operator depends of the problem type (combinatorial / numerical)

  • parent_selector – Optional – {ParentSelector} The default operator depends of the number of solution required by the genetic operator

  • mating_pool_selector – Optional – {MatingPoolSelector} The default selector is {ClosestNeighborsSelector}

  • number_of_objective – Deprecated – {integer} number of objective in the problem. Deprecated, remove in the next major release.
